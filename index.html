IMVIA Creative

Privacy Policy

Last updated: May 31, 2026

IMVIA Creative is a browser extension that connects supported Web creative tools with Adobe Photoshop through a local IMVIA Creative Bridge application. The product is designed for user-initiated local transfer of creative content.

Single Purpose

IMVIA Creative's single purpose is to let users transfer creative images or design content they actively select between supported Web creative tools, Adobe Photoshop, and the local IMVIA Creative Bridge.

The extension does not automatically scrape webpages, bypass third-party website login, payment, upload, or usage limits, or upload user creative files to IMVIA servers.

Data We Process

IMVIA Creative may process the following data only when needed to provide the extension's approved functionality:

  • User-selected images or creative content from supported websites, processed locally for transfer.
  • Current tab information needed to identify whether the active page is a supported creative tool.
  • Extension settings, language preference, feature switches, UI state, license status, and local Bridge connection status.
  • Hashed device identifier generated by the local Bridge for license binding, 7-day trial eligibility, and abuse-prevention checks.
  • Basic user activity related to the extension, such as license activation or verification status, connection diagnostics, request identifiers, timestamps, supported-site capability status, and rate-limit counters.

Local Processing Of Creative Content

User images, PSD data, SVG data, layer data, and creative files are transferred locally between the browser, the local IMVIA Creative Bridge, and Photoshop. These creative files are not uploaded to IMVIA servers.

The extension may temporarily hold user-selected content in browser memory, local extension state, or the local Bridge only for the current transfer workflow. IMVIA does not use this content for advertising, profiling, model training, or resale.

Server Communication

IMVIA servers are used only for license activation, license verification, 7-day trial eligibility, device binding, basic usage status checks, and abuse-prevention rate limiting. The server does not receive or store user creative files.

Server requests may include limited technical metadata such as a hashed device identifier, license status, request timestamp, request ID, application version, client type, and basic rate-limit counters. This information is used to provide licensing, trial eligibility, security, diagnostics, and service reliability.

Device Identifier And Trial Eligibility

To support license verification, device binding, one 7-day trial per device, and basic abuse prevention, IMVIA uses a device identifier generated by the local Bridge. Before it is sent to the License API, the identifier is hashed.

IMVIA does not upload hardware serial numbers, MAC addresses, full machine details, or raw device identifiers. The hashed device identifier is used only for licensing, trial eligibility, and abuse prevention. It is not used for advertising tracking and is not shown in the UI or logs.

Diagnostics And Logs

IMVIA Creative may generate diagnostic information to help users and support staff understand connection, authorization, Bridge, Photoshop, and supported-site adapter status. Diagnostic information is designed to be minimal and redacted.

Diagnostics do not include full license keys, full license tokens, passwords, cookies, login credentials, complete creative files, image base64 payloads, PSD data, SVG data, or webpage account tokens.

Data We Do Not Collect

IMVIA Creative does not collect:

  • Passwords.
  • Cookies or login tokens.
  • Payment information.
  • Browsing history.
  • Health information.
  • Personal communications.
  • Precise location.
  • User creative files on IMVIA servers.
  • Hardware serial numbers, MAC addresses, or raw device identifiers.

Third-Party Websites

IMVIA Creative does not bypass third-party website login, payment, upload, or usage limits. The extension only works when the user actively chooses to transfer content, import content, or connect the local Bridge and Photoshop.

Data Sharing

We do not sell user data. We do not transfer user data to third parties except as necessary to provide the extension's approved functionality, comply with law, prevent abuse, or protect the service.

We do not use user data for purposes unrelated to IMVIA Creative's single purpose, and we do not use or transfer user data to determine creditworthiness or for lending purposes.

Retention

Local settings and license state remain on the user's device until the user clears extension data or uninstalls the product. Server-side license, trial, and abuse-prevention records are retained only as long as reasonably necessary to provide licensing, security, diagnostics, and service reliability.
